# 타워 세워 (Tower Defense)

몰려오는 웨이브를 막으며 최종 스테이지까지 버티면 승리하는 타워 디펜스 게임 제작 프로젝트이다.

- 개발 기간: 2026-01-28 ~ 2026-02-03(총 5일: 개발 4일 + 안정화 1일)
- 개발 인원: 6명
- 엔진: Unity 2022.3.62f2(LTS)이다.
- 플랫폼: Windows(PC)이다.
- 입력: 키보드/마우스이다.

## 프로젝트 개요
본 프로젝트는 5일 내 완성 가능한 규모로 팀 협업, 통합, 품질 확보를 경험하기 위한 과제이다.  
목표는 "완성 가능한 결과물"을 정해진 시간에 안정적으로 제출하는 것이다.

---

## 주요 기능
- 웨이브 기반 적 스폰 및 진행 시스템이다.
- 타워 설치/전투 루프(설치 → 공격 → 처치 → 보상)이다.
- UI/HUD(웨이브/자원/상태 표시)이다. 
- 게임 상태(시작/일시정지/승리/패배) 흐름이다.
- 사운드/이펙트 최소 적용이다.
- [추가 게임 정보](Game_Info.md)
---

## 추가 기능 구현
- 캐릭터의 일정 범위 주변에 있는 적의 체력 UI 표시 (미구현)
- 준비시간 단계에서 즉시 웨이브 시작으로 진행할 수 있는 버튼 (미구현)
  - 즉시 웨이브 시작시 남은 시간에 비례한 추가 골드 지급
- 성능, 기능이 다른 타워(미구현)
- 몬스터 처치시 일정 확률 or 일정 수의 몬스터 처치시 아이템 지급 (미구현)
  - 캐릭터 주변의 몬스터 이속 저하 아이템
  - 캐릭터 주변의 몬스터에게 데미지 아이템
  - 캐릭터 주변 타워의 스탯이 일정 시간동안 증가 아이템
  - 캐릭터 체력 회복 아이템

## 조작 방법
- 기본 조작 : 키보드/마우스 기반이다.
- 캐릭터 이동 : WASD
- 타워 상호작용 : F
- 일시정지 : ESC
- UI 선택 : 마우스

---


## 폴더 구조
- `Assets/Scenes` : 씬 파일이다.
- `Assets/Prefabs` : 프리팹이다.
- `Assets/Scripts` : 스크립트 코드이다.
- `Assets/Art` : 스프라이트/모델/머티리얼이다.
- `Assets/Audio` : BGM/SFX이다.
- `Assets/ScriptableOjbect` : ScriptableObject 등 데이터 에셋이다.

---

## 협업 규칙
- 문제가 생기면 숨기지 않고 즉시 공유한다.
- 30분 이상 막히면 바로 공유한다.
- 빌드는 1일 1회 반드시 한다.
- 개인 씬 안에서만 작업한다(공용 씬 수정은 담당자 기준으로 진행한다).
- 프리팹 수정이 필요하면 복사본 생성 후 작업한다.
- 병합 충돌 발생 시 혼자 해결하지 않고 즉시 공유한다.

---

## 브랜치 전략
- `main` : 배포/빌드용 브랜치이다.
- `Develop_Branch` : 작업물 취합용 브랜치이다.
- `personal/*` 또는 개인 브랜치 : 개인 작업용 브랜치이다.
- 작업 후 `develop`으로 PR을 올리고, 머지 후 브랜치를 삭제한다.
- PR은 1일 1회 이상을 원칙으로 한다.

---

## 커밋 규칙
- 커밋 메시지는 작업 내용이 확인 가능하도록 작성한다.
- 서로 다른 성격의 복수 변경을 한 번에 커밋하지 않는다(작업 단위로 분리한다).
- 권장 형식 예시는 아래와 같다.
  - Create : 새로운 문서/ 파일 생성
  - Update : 문서/ 파일 업데이트
  - Feat : 기능 추가
  - Fix : 버그 수정
  - Test : 테스트 코드

## 그라운드 룰
### 커뮤니케이션 규칙
- *프로젝트 기간 중에 여자친구랑 연락 하지 않는다.* 
- 다른 인원이 공유한 내용 반드시 확인여부 기재한다.
- 팀원끼리 질문하거나 피드백할 때, 감정적이거나 날카롭게 굴지 않는다.
- 본인이 민폐라고 생각하지 않는다.

### GIT 버전관리 규칙
- *Main에 Push 하면 반성문 써온다.*
- 빌드는 1일 1회 반드시 한다.
- 수시로 Fetch 되었는지 확인한다.
- 개인 씬 안에서만 작업한다.
- 프리팹 수정 필요 시 복사본 생성 후 작업한다.
- 메타파일의 수정이 필요한 경우 반드시 컨펌 받고 실행한다.
- 리퀘스트 시 베이스 반드시 확인한다.
- branch 운용 방식
- 일 1회 Pull Request
- main branch : 배포 / 빌드용 
- develop branch : 작업물 취합용
- personal develop branch : 개인 작업용
- branch 명 : [이름] 작업 담당 내용
- 작업 후 develop branch로 병합
- 병합 후엔 브랜치 삭제
- 브랜치 병합 충돌시 혼자 고민 하지 말고 팀원과 공유한다.

## 팀 구성 및 역할
- 팀장 : 박승훈
  - 담당 : 데이터·툴·빌드
- 팀원 : 김태욱, 제갈도원, 최동훈, 나현수, 최원탁  
  - 담당 
    - 김태욱 : 적 스폰/웨이브
    - 제갈도원 : UI / 메뉴 / HUD
    - 최동훈 : 사운드 / 이펙트 / 연출
    - 나현수 : 타워 / 맵
    - 최원탁 : 플레이어 조작
